## 1. 问题描述 (Problem Statement)

现代社会中，越来越多的独居人士、老年人和需要情感陪伴的用户缺乏实体化的陪伴对象。现有陪伴机器人多为轮式或双足设计，在复杂地形适应性、稳定性、交互自然度方面存在局限。用户需要一款能够：
- 在家庭环境中稳定移动（不依赖平坦地面）
- 通过自然方式与用户互动（语音、动作、触摸）
- 长期学习并适应用户习惯和偏好的智能陪伴设备

**问题影响**：
- 目标用户群体（独居青年、空巢老人）缺乏情感陪伴，影响心理健康
- 现有产品交互方式单一，无法提供自然、拟人的陪伴体验
- 用户行为学习能力不足，产品无法随使用时间提升个性化体验

## 2. 产品目标 (Goals)

### 用户目标
1. **稳定移动**: 用户期望机器人能在家庭环境中稳定行走，适应不同地面材质（木地板、地毯、瓷砖）
2. **自然交互**: 用户期望通过语音、触摸、视觉多种方式与机器人互动，获得拟人的陪伴体验
3. **个性化陪伴**: 用户期望机器人能学习并记住自己的习惯、偏好，提供定制化互动
4. **情感连接**: 用户期望机器人具有可爱、亲和的外形，建立情感纽带

### 业务目标
1. **技术验证**: 完成六足步态控制、多模态交互、本地AI学习的核心技术验证
2. **MVP交付**: 在3个月内交付可运行的原型机，验证核心功能可行性
3. **用户反馈**: 收集早期用户反馈，为后续产品迭代提供数据支持

### 成功指标
- **核心功能实现率**: MVP阶段核心功能（行走、基础动作、跟踪、语音）实现率达到80%
- **稳定性指标**: 连续行走10米不摔倒，基础动作成功率≥90%
- **用户满意度**: 早期用户满意度评分≥4.0/5.0
- **技术里程碑**: 完成本地大模型部署，行为学习算法初步验证

## 3. 非目标 (Non-Goals)

本版本（V1.0 MVP）**不会**实现以下功能：

1. **复杂地形适应**: 不支持楼梯、陡坡（>10°）、严重不平整地面行走
   - *原因*: 技术复杂度高，MVP阶段优先保证平坦地面稳定性

2. **精细操作**: 不支持复杂物体操控（如拿取易碎品、书写、操作复杂工具）
   - *原因*: 末端执行器精度有限，V1.0聚焦简单抓取和基础动作

3. **云端功能**: 不支持云端数据存储、远程控制、多设备联动
   - *原因*: 隐私优先策略，所有数据本地处理，后续版本考虑可选云服务

4. **多用户识别**: 不支持多用户同时识别与个性化服务
   - *原因*: MVP阶段聚焦单一主用户场景，降低系统复杂度

5. **情感深度模拟**: 不支持深度情感理解、心理辅导等高级功能
   - *原因*: 本地大模型能力有限，V1.0聚焦基础行为学习和简单对话

## 4. 用户故事 (User Stories)

### 4.1 核心用户故事（P0 - 必须实现）

**US1: 稳定行走**
- **作为** 用户
- **我希望** 机器人能在平坦地面上稳定行走（3足或4足模式）
- **以便** 它可以跟随我在不同房间移动，提供陪伴

**验收标准**:
- [ ] 机器人支持6足行走、3足行走模式（抬起3足）和4足行走模式（抬起2足）
- [ ] 在平坦地面（木地板、瓷砖）连续行走10米不摔倒
- [ ] 行走速度可调（慢速0.2m/s、常速0.5m/s）
- [ ] 支持基本转向（左转、右转、原地旋转）

**US2: 基础动作交互**
- **作为** 用户
- **我希望** 机器人能做简单动作（招手、举手、抓取）
- **以便** 交互更生动、更有趣

**验收标准**:
- [ ] 实现单手招手动作（左前触手或右前触手）
- [ ] 实现双手同时举起和放下动作
- [ ] 左右前触手三指末端执行器可开合，抓取≤200g物品
- [ ] 动作流畅自然，无卡顿或异常震动

**US3: 主人跟踪**
- **作为** 用户
- **我希望** 机器人能识别并跟随我移动
- **以便** 它始终在我附近，随时提供陪伴

**验收标准**:
- [ ] 通过RGB-D相机检测并识别主人
- [ ] 保持1-2米跟随距离，平滑移动
- [ ] 跟踪丢失后（如遮挡）能在5秒内重新识别
- [ ] 跟踪成功率≥80%（正常光照、无严重遮挡）

**US4: 语音交互**
- **作为** 用户
- **我希望** 能通过语音与机器人对话
- **以便** 交互更自然、更便捷

**验收标准**:
- [ ] 支持离线语音识别（ASR），识别10+常见指令（"过来"、"招手"、"停止"等）
- [ ] 支持离线语音合成（TTS），回复自然流畅
- [ ] 指令识别准确率≥90%，响应延迟≤1秒
- [ ] 支持简单问答（时间、日期、天气等）

### 4.2 重要用户故事（P1 - 应该实现）

**US5: 洛基外形设计**
- **作为** 用户
- **我希望** 机器人外形可爱、亲和
- **以便** 建立情感连接，愿意长期与之互动

**验收标准**:
- [ ] 外形高度还原洛基特征（六足、触手、头部造型）
- [ ] 还原度≥80%（用户调研评分）
- [ ] 材质安全无毒，适合家庭环境
- [ ] 尺寸适中（高30-50cm），不占用过多空间

**US6: 行为学习（基础）**
- **作为** 用户
- **我希望** 机器人能逐渐学习我的习惯和偏好
- **以便** 互动越来越符合我的期望

**验收标准**:
- [ ] 本地部署轻量级大模型（LLaMA 3 8B量化版本）
- [ ] 记录用户交互数据（指令习惯、活动规律）
- [ ] 每周在闲置时段进行增量学习，优化响应
- [ ] 用户可感知行为变化（如主动问候时间、记住常用指令）

### 4.3 未来用户故事（P2 - 后续版本）

**US7: 情感交互**
- **作为** 用户
- **我希望** 机器人能识别我的情绪状态并做出相应反应
- **以便** 获得情感支持和陪伴

**US8: 多模态感知**
- **作为** 用户
- **我希望** 机器人能通过触摸、声音、视觉多种方式感知我的需求
- **以便** 交互更丰富、更智能

**US9: 自主导航**
- **作为** 用户
- **我希望** 机器人能自主建图并在室内自由移动
- **以便** 它能在多房间环境中主动找到我

## 5. 需求说明 (Requirements)

### 5.1 功能需求

#### 5.1.1 移动与步态控制（P0 - 必须实现）

| 需求ID | 需求描述 | 验收标准 | 技术考虑 |
|--------|----------|----------|----------|
| REQ-002 | 支持6足静态行走 | 行走速度0.2-0.5m/s | 优化的步态算法，减少机身晃动 |
| REQ-001 | 支持3足静态行走 | 抬起2足，其余3足支撑，连续行走10米不摔倒 | 使用PyBullet仿真验证步态稳定性，采用静态步态规划（单足摆动，其余支撑） |
| REQ-002 | 支持4足静态行走 | 抬起1足，其余4足支撑，行走速度0.2-0.5m/s | 优化的步态算法，减少机身晃动 |
| REQ-003 | 支持基本转向 | 支持左转、右转、原地旋转，转向误差≤15° | 差速转向或调整各足步长实现转向 |
| REQ-004 | 电池续航 | 连续工作时间≥1小时（行走+交互） | 采用2S-3S锂聚合物电池，容量≥5000mAh |

#### 5.1.2 动作执行（P0 - 必须实现）

| 需求ID | 需求描述 | 验收标准 | 技术考虑 |
|--------|----------|----------|----------|
| REQ-005 | 单手招手动作 | 左前或右前触手完成招手动作，重复5次成功 | 触手3自由度设计，预设招手轨迹 |
| REQ-006 | 双手举放动作 | 双前触手同步举起和放下，动作流畅 | 协调控制算法，保证双触手同步 |
| REQ-007 | 三指抓取动作 | 前触手末端执行器可开合，抓取≤200g物品 | 3指并联机构，采用微型舵机驱动 |
| REQ-008 | 翘指动作 | 单指独立翘起（如竖大拇指） | 每指独立驱动，预设翘指轨迹 |

#### 5.1.3 视觉感知与跟踪（P0 - 必须实现）

| 需求ID | 需求描述 | 验收标准 | 技术考虑 |
|--------|----------|----------|----------|
| REQ-009 | 人体检测 | 基于RGB-D相机（Intel RealSense D435i）检测人体 | 使用YOLOv8或类似模型，本地推理 |
| REQ-010 | 主人识别 | 通过人脸或体态特征识别主人（MVP阶段简化为人形检测） | MVP阶段不实现人脸识别，仅跟踪最近人形目标 |
| REQ-011 | 视觉跟踪 | 保持1-2米距离跟随主人，跟踪成功率≥80% | PID控制算法，根据目标位置调整移动速度 |
| REQ-012 | 遮挡处理 | 跟踪丢失后5秒内重新识别 | 记录目标特征，遮挡结束后快速重新检测 |

#### 5.1.4 语音交互（P0 - 必须实现）

| 需求ID | 需求描述 | 验收标准 | 技术考虑 |
|--------|----------|----------|----------|
| REQ-013 | 离线语音识别 | 支持10+常见指令识别，准确率≥90% | 使用Vosk离线ASR引擎，自定义指令词库 |
| REQ-014 | 离线语音合成 | 支持中文语音回复，响应延迟≤1秒 | 使用Edge TTS或类似离线TTS引擎 |
| REQ-015 | 简单问答 | 支持时间、日期、天气等简单问题回答 | 接入本地知识库或轻量级QA模型 |
| REQ-016 | 唤醒词检测 | 通过唤醒词（如"洛基"）激活语音交互 | 使用Porcupine或类似唤醒词引擎 |

#### 5.1.5 行为学习与个性化（P1 - 应该实现）

| REQ-019 | 增量学习 | 每周闲置时段进行模型微调，优化响应 | 使用LoRA或类似参数高效微调方法 |

| REQ-023 | 尺寸适中 | 高度30-50cm，适合家庭场景 | 结构优化，在保证功能前提下减小体积 |

### 9.2 术语表

| 术语 | 定义 |
|------|------|
| 静态步态 | 行走过程中始终有≥3足支撑地面的步态，稳定性高但速度慢 |
| 动态步态 | 行走过程中可能出现腾空相（无足支撑）的步态，速度快但稳定性要求高 |
| RGB-D相机 | 同时输出彩色图像（RGB）和深度信息（D）的相机 |
| 增量学习 | 在已有模型基础上，用新数据继续训练，无需重新训练整个模型 |
| LoRA | Low-Rank Adaptation，一种参数高效的微调方法，适合资源受限场景 |
